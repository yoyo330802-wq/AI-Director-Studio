## 定价结构

### 1. 按需计费 (On-Demand)

#### Studio 界面显示价格 (元/秒)

| 模式 | 模式ID | 模型 | 单价 | 预估时间 | 最大时长 |
|------|--------|------|------|----------|----------|
| 闪电模式 | `fast` | Wan2.1-1.3B | ¥0.04/秒 | ~15秒 | 5秒 |
| 智能模式 | `balanced` | 智能路由 | ¥0.06/秒 | ~30秒 | 10秒 |
| 专业模式 | `premium` | Vidu/可灵 | ¥0.09/秒 | ~60秒 | 10秒 |

#### 定价计算公式

```
费用 = 视频时长(秒) × 单价(元/秒)
Token消耗 = 费用 × 100 (1元 = 100 Token)
```

#### 示例计算

| 时长 | 闪电模式 | 智能模式 | 专业模式 |
|------|----------|----------|----------|
| 5秒 | ¥0.20 / 20 Tokens | ¥0.30 / 30 Tokens | ¥0.45 / 45 Tokens |
| 10秒 | ¥0.40 / 40 Tokens | ¥0.60 / 60 Tokens | ¥0.90 / 90 Tokens |

## 图生视频 (Image-to-Video) 定价

### 规则

- **图生视频仅支持专业模式 (premium)**
- 单价与专业模式一致: ¥0.09/秒
- 必须提供 `image_url` 参考图片

### API 要求

```json
POST /api/v1/generate
{
    "prompt": "视频描述",
    "duration": 5,
    "quality_mode": "quality",  // 必须用 quality
    "image_url": "https://example.com/reference.jpg"
}
```

### 路由自动选择

当 `image_url` 存在时:
- 智能路由自动选择 Vidu 或 可灵
- 不会路由到 Wan2.1（不支持图生视频）

## 常见问题

### Q: 按需计费和套餐可以同时使用吗？

A: 可以。用户可以同时拥有套餐分钟数和Token余额。生成任务时:
1. 优先扣除套餐分钟数
2. 套餐用完后，使用Token余额按需计费

### Q: 为什么专业模式比智能模式贵？

A: 专业模式使用 Vidu/可灵等商业级模型，成本更高:
- 智能模式使用自建 Wan2.1，成本 ¥0.012-0.025/秒
- 专业模式使用硅基流动 API，成本 ¥0.050-0.070/秒

### Q: 图生视频为什么要强制专业模式？

A: 只有 Vidu 和 可灵 支持图生视频功能，Wan2.1 自建集群不支持。

### Q: 如何选择最经济的方案？

A: 使用 `/api/v1/bill/on-demand/compare` API 对比:
- 偶尔使用: 按需计费更划算
- 长期高频: 套餐更划算
